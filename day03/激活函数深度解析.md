## 1. 为什么需要激活函数？

### 1.1 线性变换的局限性

假设我们有一个没有激活函数的神经网络，它实际上只是线性变换的复合：

```
输出 = Wn × (Wn-1 × (... × (W2 × (W1 × 输入))))
     = W' × 输入
```

无论网络有多少层，**没有激活函数的深度神经网络等价于单层线性模型**！这意味着：

- ❌ 无法学习非线性关系
- ❌ 无法解决 XOR 等简单问题
- ❌ 表达能力极其有限

### 1.2 激活函数引入非线性

激活函数是神经网络的"灵魂"，它：

- ✅ 引入非线性变换，使网络能够学习复杂的非线性关系
- ✅ 赋予神经网络强大的表达能力
- ✅ 是深度学习能够成功的基础

**直观理解**：激活函数就像神经元的"开关"，决定神经元是否被"激活"（传递信息）。

## 2. 激活函数的核心作用

### 2.1 数学角度

激活函数 $f(x)$ 使网络能够拟合任意复杂的函数：

$$y = f(W \cdot x + b)$$

如果没有 $f$，这就是简单的线性变换。

### 2.2 几何角度

| 激活函数 | 几何意义 |
|---------|---------|
| 无 | 超平面（直线） |
| Sigmoid | S形曲线 |
| ReLU | 折线 |
| 多个ReLU组合 | 可以逼近任意凸函数 |

### 2.3 神经科学角度

激活函数模拟了生物神经元的"全有或全无"（all-or-none）特性：

- 当输入超过阈值时，神经元"激活"
- 否则保持"静默"

## 3. 常用激活函数详解

### 3.1 Sigmoid（S形函数）

#### 数学公式

$$\sigma(x) = \frac{1}{1 + e^{-x}}$$

#### 导数

$$\sigma'(x) = \sigma(x) \cdot (1 - \sigma(x))$$

#### 图像特征

```
         1 ─────────────────
        /|
       / |
      /  |
  0.5───┼──────
     /  |
    /   |
   /    |
  0 ────┼────────
    -5   0   +5
```

#### 优缺点分析

| 优点 | 缺点 |
|------|------|
| ✓ 输出范围 (0,1)，表示概率方便 | ✗ 梯度消失问题（x很大或很小时梯度接近0） |
| ✓ 光滑、可导 | ✗ 输出非零中心化（影响收敛速度） |
| ✓ 适合二分类问题 | ✗ 指数运算计算较慢 |

#### 适用场景

- ✅ 二分类问题的**输出层**
- ✅ 需要概率输出的场景
- ❌ 不推荐用于深层网络的**隐藏层**

---

### 3.2 Tanh（双曲正切函数）

#### 数学公式

$$\tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}$$

#### 导数

$$\tanh'(x) = 1 - \tanh^2(x)$$

#### 图像特征

```
       1 ────
       |\
       | \
  0.5──┼──┼──────
       |  \
  0.0──┼───┼────
       |    \
      -1     \
```

#### 优缺点分析

| 优点 | 缺点 |
|------|------|
| ✓ 输出范围 (-1,1)，零中心化 | ✗ 仍有梯度消失问题 |
| ✓ 收敛速度比 Sigmoid 快 | ✗ 指数运算计算较慢 |

#### 适用场景

- ✅ 适合循环神经网络（RNN、LSTM、GRU）
- ✅ 适合隐藏层（比 Sigmoid 更好）
- ❌ 不适合输出层（除非需要 (-1,1) 输出）

---

### 3.3 ReLU（线性整流单元）

#### 数学公式

$$ReLU(x) = \max(0, x) = \begin{cases} x, & x > 0 \\ 0, & x \leq 0 \end{cases}$$

#### 导数

$$ReLU'(x) = \begin{cases} 1, & x > 0 \\ 0, & x \leq 0 \end{cases}$$

#### 图像特征

```
         |\
         | \
         |  \
    0────┼───┼──────
         |
         |
```

#### 优缺点分析

| 优点 | 缺点 |
|------|------|
| ✓ 计算简单快速（无指数运算） | ✗ 死神经元问题（ReLU死亡） |
| ✓ 解决梯度消失问题（正区间梯度恒为1） | ✗ 输出非零中心化 |
| ✓ 稀疏激活（约50%神经元被抑制） | ✗ 不可微（在x=0处） |

#### 什么是"ReLU死亡"？

当某个神经元的输入始终为负，其梯度始终为0，权重无法更新，该神经元"死亡"。

**原因**：
- 学习率过大
- 负偏置初始化
- 大量负输入数据

#### 适用场景

- ✅ **首选**的隐藏层激活函数
- ✅ 卷积神经网络（CNN）
- ✅ 深度前馈网络

---

### 3.4 LeakyReLU（带泄漏的ReLU）

#### 数学公式

$$LeakyReLU(x) = \begin{cases} x, & x > 0 \\ \alpha \cdot x, & x \leq 0 \end{cases}$$

其中 $\alpha$ 是很小的正数（通常取 0.01 或 0.1）

#### 图像特征

```
         |\
         | \
     ____|  \______
      /      \
     /        \
```

#### 优缺点分析

| 优点 | 缺点 |
|------|------|
| ✓ 解决 ReLU 死亡问题 | ✗ $\alpha$ 是超参数，需要调整 |
| ✓ 保持 ReLU 的优点 | ✗ 负区间有少量梯度流出 |

#### 适用场景

- ✅ 出现大量死神经元时
- ✅ 需要 ReLU 但担心死亡问题时

---

### 3.5 ELU（指数线性单元）

#### 数学公式

$$ELU(x) = \begin{cases} x, & x > 0 \\ \alpha \cdot (e^x - 1), & x \leq 0 \end{cases}$$

#### 优缺点分析

| 优点 | 缺点 |
|------|------|
| ✓ 输出接近零中心化 | ✗ 指数运算，计算较慢 |
| ✓ 负区间有平滑梯度 | ✗ 超参数 $\alpha$ 需要调整 |

#### 适用场景

- ✅ 需要零中心化输出的网络
- ✅ 对精度要求较高的场景

---

### 3.6 GELU（高斯误差线性单元）

#### 数学公式

$$GELU(x) = x \cdot \Phi(x)$$

其中 $\Phi(x)$ 是标准正态分布的累积分布函数（CDF）

近似计算：
$$GELU(x) \approx 0.5 \cdot x \cdot (1 + \tanh(\sqrt{\frac{2}{\pi}} \cdot (x + 0.044715 \cdot x^3)))$$

#### 优缺点分析

| 优点 | 缺点 |
|------|------|
| ✓ 平滑且非零中心化 | ✗ 计算复杂度较高 |
| ✓ 性能优于 ReLU 和 LeakyReLU |  |
| ✓ Transformer模型的标准选择 |  |

#### 适用场景

- ✅ **Transformer架构**（GPT、BERT、ViT等）
- ✅ 最先进的自然语言处理模型

---

### 3.7 Swish（自门控激活函数）

#### 数学公式

$$Swish(x) = x \cdot \sigma(x) = x \cdot \frac{1}{1 + e^{-x}}$$

#### 优缺点分析

| 优点 | 缺点 |
|------|------|
| ✓ 平滑、无界、非单调 | ✗ 计算比 ReLU 稍慢 |
| ✓ 在某些任务上表现优异 |  |
| ✓ 自门控机制 |  |

#### 适用场景

- ✅ 需要更高精度的场景
- ✅ 深度残差网络

---

### 3.8 Softmax（归一化指数函数）

#### 数学公式

$$Softmax(x_i) = \frac{e^{x_i}}{\sum_{j=1}^{K} e^{x_j}}$$

#### 特点

- ✅ 输出总和为 1
- ✅ 适合多分类问题的**输出层**
- ✅ 每个输出可以理解为概率

#### 适用场景

- ✅ **多分类问题的输出层**
- ✅ 注意力机制（Transformer）
- ❌ 不用于隐藏层

## 4. 激活函数选择指南

### 4.1 根据网络类型选择

| 网络类型 | 推荐激活函数（隐藏层） | 推荐激活函数（输出层） |
|---------|---------------------|---------------------|
| **CNN** | ReLU / LeakyReLU | Sigmoid（二分类）<br>Softmax（多分类） |
| **MLP** | ReLU / GELU | Sigmoid / Softmax |
| **RNN/LSTM** | Tanh / ReLU | Tanh（有时） |
| **Transformer** | GELU | - |

### 4.2 根据问题类型选择

| 问题类型 | 输出层激活函数 |
|---------|---------------|
| 二分类 | Sigmoid |
| 多分类 | Softmax |
| 回归 | Linear（无激活函数） |
| 多标签分类 | Sigmoid（每个标签独立） |

### 4.3 实践经验总结

```
默认选择：
  └── ReLU（隐藏层）

遇到问题：
  ├── 死神经元太多 → LeakyReLU
  ├── 梯度消失 → GELU / Swish
  ├── Transformer架构 → GELU
  ├── RNN/LSTM → Tanh
  └── 需要概率输出 → Sigmoid / Softmax
```

## 5. 实践建议与常见问题

### 5.1 梯度消失问题

**问题**：深层网络中，梯度逐层衰减，无法更新早期层参数。

**原因**：
- Sigmoid/Tanh 的导数最大值为 1，大部分情况 < 0.25
- 多层相乘后梯度趋近于 0

**解决方案**：
1. ✅ 使用 ReLU（正区间梯度恒为1）
2. ✅ 使用残差连接（ResNet）
3. ✅ 使用 GELU 等现代激活函数

### 5.2 死神经元问题

**问题**：ReLU 神经元对负输入输出恒为0，梯度为0，无法激活。

**检测方法**：
```python
# 统计死神经元比例
dead_ratio = (activations == 0).float().mean()
```

**解决方案**：
1. ✅ 使用 LeakyReLU 或 ELU
2. ✅ 降低学习率
3. ✅ 使用更好的初始化方法（如 He 初始化）

### 5.3 参数初始化建议

| 激活函数 | 推荐初始化方法 |
|---------|---------------|
| ReLU / LeakyReLU | He 初始化 |
| Sigmoid / Tanh | Xavier (Glorot) 初始化 |
| 其他 | Xavier 初始化 |

**PyTorch 实现**：
```python
# He 初始化（适合 ReLU）
nn.init.kaiming_normal_(layer.weight, mode='fan_in', nonlinearity='relu')

# Xavier 初始化（适合 Sigmoid/Tanh）
nn.init.xavier_normal_(layer.weight)
```

### 5.4 批归一化（Batch Normalization）

激活函数通常配合批归一化使用：

```
输入 → BN → 激活函数 → ...
```

**好处**：
- ✅ 稳定训练
- ✅ 允许更大的学习率
- ✅ 减少对初始化的敏感度

## 6. PyTorch代码实现

### 6.1 基本用法

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

# 方法1：使用 nn.Module（推荐）
class Model(nn.Module):
    def __init__(self):
        super().__init__()
        self.relu = nn.ReLU()
        self.leaky_relu = nn.LeakyReLU(0.1)
        self.elu = nn.ELU(alpha=1.0)
        self.gelu = nn.GELU()

    def forward(self, x):
        x = self.relu(x)
        return x

# 方法2：使用函数式 API
x = torch.randn(10, 10)
x = F.relu(x)                # ReLU
x = F.leaky_relu(x, 0.1)     # LeakyReLU
x = F.elu(x, alpha=1.0)      # ELU
x = F.gelu(x)                # GELU
x = torch.sigmoid(x)         # Sigmoid
x = torch.tanh(x)            # Tanh
x = F.softmax(x, dim=1)      # Softmax
```

### 6.2 完整网络示例

```python
class SimpleCNN(nn.Module):
    def __init__(self):
        super().__init__()
        self.conv1 = nn.Conv2d(3, 64, 3, padding=1)
        self.bn1 = nn.BatchNorm2d(64)
        self.conv2 = nn.Conv2d(64, 128, 3, padding=1)
        self.bn2 = nn.BatchNorm2d(128)
        self.fc = nn.Linear(128 * 8 * 8, 10)
        self.pool = nn.MaxPool2d(2)

    def forward(self, x):
        # Conv → BN → ReLU → Pool
        x = self.pool(F.relu(self.bn1(self.conv1(x))))
        x = self.pool(F.relu(self.bn2(self.conv2(x))))
        x = x.view(x.size(0), -1)
        x = self.fc(x)  # 配合 CrossEntropyLoss（内置 Softmax）
        return x
```

### 6.3 自定义激活函数

```python
class CustomActivation(nn.Module):
    def __init__(self):
        super().__init__()

    def forward(self, x):
        # 例如：Swish
        return x * torch.sigmoid(x)

# 或使用函数
def custom_activation(x):
    return x * torch.sigmoid(x)
```

## 总结

| 激活函数 | 核心特点 | 最佳场景 |
|---------|---------|---------|
| **Sigmoid** | (0,1)输出，适合概率 | 二分类输出层 |
| **Tanh** | (-1,1)输出，零中心化 | RNN/LSTM |
| **ReLU** | 简单高效，解决梯度消失 | CNN隐藏层（首选）|
| **LeakyReLU** | 解决死神经元 | 有死ReLU问题时 |
| **ELU** | 平滑，零中心化 | 高精度场景 |
| **GELU** | Transformer标准选择 | NLP/Transformer |
| **Swish** | 自门控，性能优异 | 深度残差网络 |
| **Softmax** | 概率分布，总和为1 | 多分类输出层 |

**记住**：
1. 🎯 隐藏层默认用 **ReLU**
2. 🎯 Transformer 用 **GELU**
3. 🎯 多分类输出用 **Softmax**
4. 🎯 二分类输出用 **Sigmoid**
